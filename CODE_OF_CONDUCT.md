# Code Of Conduct

Portolan uses lightweight conduct guidelines based on Contributor Covenant
principles, scoped for a small maintainer-led open-source project.

## Expected Behavior

- Be direct, specific, and respectful.
- Keep discussion grounded in code, docs, tests, evidence, and product
  boundaries.
- Preserve uncertainty labels such as `unknown`, `cannot_verify`,
  `not_assessed`, `blocked`, and `failed`.
- Assume public threads may be read by future contributors and users.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private source, secrets, customer data, or sensitive vulnerability
  details.
- Repeatedly reframing unsupported claims as verified.
- Deliberately bypassing repository safety, evidence, or review rules.

## Enforcement

Maintainers may edit, hide, close, or lock comments, issues, and pull requests
that violate these guidelines. Maintainers may also block users who repeatedly
ignore the project boundary or create unsafe public disclosures.

For sensitive vulnerability reports, use [SECURITY.md](SECURITY.md). For
non-sensitive conduct concerns, open a GitHub issue with only the public details
needed for maintainers to act. Until a dedicated private conduct contact exists,
do not post sensitive personal details publicly.

If you believe an enforcement action was applied in error, open a good-faith
public issue with non-sensitive details so maintainers can review it.
