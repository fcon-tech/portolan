# Spec Delta — reading-experience

## ADDED Requirements

### Requirement: Published demo is generated from a real pipeline run
The published Portolan demo SHALL be generated by running the full pipeline
(scan → system-map → nav-bundle → SI → export) on a real target landscape.
The demo SHALL NOT use hand-crafted system-map fixtures in the live path.
All artifact IDs SHALL be coherent across pipeline stages.

#### Scenario: Demo atlas shows real edges from a real scan
- GIVEN the demo rebuild script runs against the Apache Bigtop landscape
- WHEN the pipeline completes
- THEN the atlas shows depends-on edges from real pom.xml/build.gradle manifests
- AND the atlas shows references edges from real JVM source imports
- AND no edge is hand-crafted

#### Scenario: All IDs are coherent across stages
- GIVEN the pipeline produces system-map, nav-bundle, and SI artifacts
- WHEN a component link in the nav-bundle is clicked
- THEN the component exists in the system-map
- AND the investigation page resolves the same component ID
