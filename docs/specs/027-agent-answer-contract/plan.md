# Implementation Plan: Agent Answer Contract

## Decision Gate

- Simpler/Faster: add one Markdown artifact to the existing context pack and
  point existing docs at it. Do not add MCP, a daemon, a new query language, or
  a second JSON schema in this slice.
- Blocking Edge Cases: agents need a stable instruction surface, but the
  context pack does not know whether a later `map` run already exists. The
  contract must therefore name the map artifacts and required evidence states
  without pretending they were generated by `context prepare`.
- Existing Open Source: this is an agent-instruction contract over local
  Portolan artifacts. OSS scanners remain producers through `tool-registry.json`
  and `oss-plan.json`; no new dependency is justified.

## Technical Approach

- Extend `internal/contextprep.Artifacts` with `AnswerContract`.
- Write `answer-contract.md` during `context prepare`.
- Render a compact, deterministic Markdown contract that:
  - starts with the current context pack artifacts;
  - tells agents when to run `portolan map --root <target-root> --out <run-dir>`;
  - maps CTO question families to evidence surfaces;
  - mandates answer fields: answer, evidence, unknowns/not_assessed, next local
    command.
- Update `agent-brief.md`, `query-plan.md`, CLI help, portable skill, Cursor
  rule, and agent docs to include the new artifact.
- Add focused tests for artifact emission and contract content.
