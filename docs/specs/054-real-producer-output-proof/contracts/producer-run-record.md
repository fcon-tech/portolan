# Contract: Producer Run Records

Producer run records are local evidence metadata supplied to context and map
artifacts. They describe outputs generated by existing tools; they do not ask
Portolan to execute those tools.

## JSONL Record Shape

```json
{
  "record_type": "producer-run",
  "id": "producer-run-bigtop-compose-20260601",
  "producer_family": "deployment-model",
  "producer_tool": "docker-compose",
  "command": "DOCKER_IMAGE=apache/bigtop-placeholder MEM_LIMIT=1g docker compose -f repos/apache-bigtop-repo/provisioner/docker/docker-compose.yml config --format json",
  "target_root": "/home/fall_out_bug/projects/bigtop-landscape",
  "output_path": ".portolan/stress/20260601-054-initial-proof/tool-outputs/apache-bigtop-compose.config.json",
  "output_format": "json",
  "scope": {
    "repository": "apache-bigtop-repo",
    "directory": "provisioner/docker",
    "covered_units": ["service:bigtop", "network:default"]
  },
  "freshness": "2026-06-01T20:25:23Z",
  "status": "verified",
  "evidence_state": "metadata-visible",
  "limitations": [
    "static deployment model only",
    "not runtime topology",
    "placeholder environment values used for config rendering"
  ],
  "privacy_review": "not_assessed"
}
```

## Required Behavior

- Unknown, missing, blocked, failed, and partial records are valid.
- Static producer outputs must not set `evidence_state` to `runtime-visible`.
- Records with `status: verified` must point to an existing local output when
  validated.
- Records must be safe to include in agent context without raw source snippets
  or credentials.

## Rejection Conditions

- `runtime-visible` evidence state for non-runtime producer families.
- Network, credential, daemon, or target mutation behavior hidden in the record.
- Output paths outside the selected root/output boundary without explicit
  operator selection.
- Positive architecture support claim with `status: not_assessed`.
