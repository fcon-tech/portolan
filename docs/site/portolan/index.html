Local-first codebase mapping

Portolan builds local evidence maps for agents.

It reads local source, metadata, runtime exports, and claim files, then writes bounded artifacts that separate visible evidence from unknown, cannot-verify, and not-assessed surfaces.

Install Run the Bigtop demo
bounded workflow read-only 
portolan context prepare \
  --root <target-root> \
  --out <output-dir>/context \
  --profile cursor

portolan map \
  --root <target-root> \
  --out <output-dir>/map

Use it before broad codebase answers

Portolan is useful when an agent needs a bounded local evidence pack before it answers architecture, dependency, duplication, configuration, or technical-debt questions.

Context pack

Agent brief, answer contract, evidence index, repository inventory, tool registry, OSS plan, and explicit gaps.

Map bundle

Run metadata, coverage, summary, graph index, graph JSON, findings JSONL, and a human-readable map.

Query surface

Bounded commands for findings and gaps so agents do not need to load the full graph before every question.

Boundaries that stay visible

The validated claim is narrow: Portolan prepares local evidence for agents and helps reduce unsupported claims in the fixed local Bigtop comparison. It does not prove broad external coverage.

Not a replacement

Portolan complements Cursor, OpenCode, Codex, Sourcegraph, Backstage, service catalogs, observability, and modernization tools.

Not complete estate proof

Repository counts and local runs show visible scope. Complete inherited-estate coverage remains unproven unless inventory evidence supports it.

Not broad security certification

The current security boundary covers selected local CLI risks and untrusted-artifact handling, not a general security audit.
