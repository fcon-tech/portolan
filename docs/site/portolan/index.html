Captain atlas for software landscapes

Give an agent Portolan and a target ecosystem. Get a local atlas.

Portolan packages the install guide, target-local wrappers, local producers, bounded query tools, and viewer app an agent needs to map a multi-repo codebase, explain relationships, and drill from the atlas into reports and source snippets.

The first screen is the product

The useful outcome is not a report template. It is a local atlas app plus a queryable bundle: repos, relationships, risks, gaps, selected-code context, and captain-facing handoff in one workflow.

Boundaries that stay visible

Portolan preserves proof boundaries, but evidence is not the headline. The headline is navigation: when a relationship, risk, gap, or answer cannot be backed by local bundle facts, it remains unknown, cannot_verify, or not_assessed.

Not a replacement

Portolan complements Cursor, OpenCode, Codex, Sourcegraph, Backstage, service catalogs, observability, and modernization tools.

Not complete estate proof by default

Repository counts and local runs show visible scope. Complete inherited-estate coverage remains unproven unless inventory evidence supports it.

Not broad security certification

The current security boundary covers selected local CLI risks and untrusted-artifact handling, not a general security audit.
