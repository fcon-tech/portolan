01

First run for agents

The product starts when a coding agent reads the Portolan guide, asks only blocking questions, installs target-local wrappers, and runs a bounded scan.

  • Cursor Composer is the first acceptance client.
  • OpenCode, Codex, Kimi/Zed-like harnesses stay portable.
  • Target source is read-only unless explicitly approved.
02

Atlas app, not report theater

The useful artifact is a local viewer plus a queryable bundle: repositories, components, package/deploy/test modules, relationships, hotspots, gaps, and handoff.

  • Map first, drill-down second.
  • Unknowns stay visible without becoming the headline.
  • Large indexes stay local and stratified.
03

Agent navigation layer

Portolan gives agents bounded commands and stable bundle contracts, so they can answer landscape questions without re-inventing discovery for every codebase.

  • Repos, relationships, risks, gaps, search, selected code.
  • Receipts and progress for first-run handoff.
  • Local-first bundle with no network requirement by default.

What a client should feel

They open an atlas, see the shape of the estate, inspect the cross-repo connective tissue, and ask the augmented coding agent to drill into a selected node or source area.
