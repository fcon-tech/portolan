Local-first codebase mapping

Portolan builds local evidence maps for agents.

It installs target-local wrappers, reads local source, metadata, runtime exports, and claim files, then writes bounded atlas artifacts that separate visible evidence from unknown, cannot_verify, and not_assessed surfaces.

Use it before broad codebase answers

Portolan is useful when an agent needs a bounded local evidence atlas before it answers architecture, dependency, duplication, configuration, or technical-debt questions.

Installable harness

Cursor rule, OpenCode AGENTS block, and target-local command wrappers under .portolan/bin.

Atlas bundle

Manifest, repository profiles, relationships, hotspots, gaps, search, symbols, claims, and source drill-down routes.

Query surface

Bounded commands for repos, relationships, hotspots, gaps, search, symbols, claims, and source snippets.

Boundaries that stay visible

The validated claim is narrow: Portolan prepares local evidence for agents and helps reduce unsupported claims in the fixed local Bigtop comparison. It does not prove broad external coverage.

Not a replacement

Portolan complements Cursor, OpenCode, Codex, Sourcegraph, Backstage, service catalogs, observability, and modernization tools.

Not complete estate proof

Repository counts and local runs show visible scope. Complete inherited-estate coverage remains unproven unless inventory evidence supports it.

Not broad security certification

The current security boundary covers selected local CLI risks and untrusted-artifact handling, not a general security audit.
