What Portolan is for

Use Portolan when an agent needs bounded local evidence before it answers architecture, dependency, duplication, configuration, or technical-debt questions across a codebase or software landscape.

Local evidence first

Tools should read local inputs and keep missing evidence visible instead of turning gaps into confident prose.

Agent-facing workflows

Outputs are designed for coding agents, staff engineers, and reviewers who need bounded context before decisions.

Claim-bounded output

Public copy and agent packets point back to product claims, release notes, demo evidence, contribution rules, and security boundaries.
