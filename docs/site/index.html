Portolan project

Portolan gives agents a local atlas before they answer.

A captain gives an agent Portolan plus a target ecosystem. The agent installs target-local wrappers, builds a local atlas app, and uses bounded query routes to explain the landscape.

What Portolan is for

Use Portolan when an agent needs to orient a captain across repositories, dependencies, duplication, configuration, risks, and gaps without loading a whole codebase into chat.

Atlas first

The primary output is a local viewer and queryable bundle, not a static consulting report.

Agent-facing workflows

Outputs are designed for Cursor, OpenCode, Codex-compatible harnesses, Claude-style notes, and shell-capable agents.

Drill-down boundaries

Answers point back to bundle records, source snippets, relationship rows, gaps, and viewer routes.
