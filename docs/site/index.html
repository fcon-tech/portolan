What FCON publishes here

Small, inspectable tools and documentation for teams using AI agents in real engineering work. The public surface starts with Portolan and stays inside verified repository evidence.

Local evidence first

Tools should read local inputs and keep missing evidence visible instead of turning gaps into confident prose.

Agent-facing workflows

Outputs are designed for coding agents, staff engineers, and reviewers who need bounded context before decisions.

Claim-bounded launches

Public copy points back to product claims, release notes, demo evidence, contribution rules, and security boundaries.
