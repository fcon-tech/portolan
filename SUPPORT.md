# Support

Portolan is an open-source local CLI maintained on a best-effort basis. There is
no public support SLA.

## Public Questions And Bugs

Use GitHub issues for:

- reproducible bugs;
- documentation gaps;
- evidence-state or artifact questions;
- focused feature proposals;
- product-claim corrections.

Include the command you ran, the target shape, the output path, and one of these
states: `verified`, `failed`, `blocked`, `unknown`, `cannot_verify`, or
`not_assessed`.

## Sensitive Security Reports

Do not put sensitive vulnerability details in public issues, pull requests, or
discussions. Use the private reporting route in [SECURITY.md](SECURITY.md).

## Boundaries

Portolan is not a managed service, readiness gate, service catalog,
observability platform, modernization engine, or coding harness. Maintainers may
close requests that depend on those unsupported scopes and point them to the
current [Product Claims](docs/product-claims.md).
